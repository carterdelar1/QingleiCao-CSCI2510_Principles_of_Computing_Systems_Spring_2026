10. 
man 2 open: error returns -1 and sets errno

man 2 close: error returns -1 and sets errno

man 2 read: on error returns -1 and sets errno

man 2 write: on error returns -1 and sets errno

man 3 printf: on error returns a negative falue. does not set errno.

11. error checking this way simplifies software development because:
it makes it easy to figure out why something might not work, because it gives an error explanation for many cases, 
and it works consistently over many different functions, so you don't have to do different error checking for different functions.
